Recommended product direction

Make desktop proof usable from a phone.

Lancer should not replace Playwright, Cypress, Maestro, Appium, Postman, k6, Storybook, or Sentry. It should turn their outputs into mobile-native engineering decisions.

The killer feature is a cockpit for Repro, Trace, Proof, Risk, and Memory. Agents run harnesses on the desktop. Lancer shows the evidence that lets a developer decide from mobile without compromise.

Each stage adds evidence. Lancer's job is to compress that evidence into a phone-sized decision, not to expose another raw terminal or desktop dashboard.

01

Intake

Screenshot, Clip, Sentry replay, failed CI, API example, or customer report becomes an Evidence Inbox item.

02

Repro

Agent creates the smallest runnable repro: Playwright, Maestro, Appium, API request, or manual steps.

03

Scope

Harness Plan defines required proof, affected areas, forbidden files, devices, browsers, and thresholds.

04

Build

Work Thread becomes harness-aware: hypotheses, attempts, failed runs, screenshots, retries, and final fix.

05

Validate

Proof Matrix shows whether the contract was actually met before the user trusts the result.

06

Review

Review claims with evidence first. Open the diff only when a proof card needs deeper inspection.

07

Ship

PR Proof Packet attaches traces, visual diffs, API results, performance status, and unresolved risks.

08

Watch

Post-ship window checks whether the original error or performance issue returns in production.

09

Memory

Regression Vault stores the repro, proof, and baseline so future agents rerun it when related files change.

What the desktop tools actually look like.

Lancer borrows patterns from tools you may not use daily. Each block below shows the real desktop UI on the left, then how Lancer compresses it for phone on the right. No jargon required.

You don't run these on your phone. Your Mac runs Playwright, Cypress, Postman, etc. Lancer reads their output files (traces, screenshots, API results) and shows you a phone-friendly version so you can approve or reject without opening a laptop.

Cypress — "Time Travel" command log

What it is: A browser testing app for websites. Its killer feature is a command log on the left — every action the test took (visit page, click button, check text) — and a live preview of the website on the right. Click any line in the log and the preview rewinds to exactly what the page looked like at that moment. That's "Time Travel."

Sentry — Error tracking + session replay

What it is: A production monitoring service. When your live app crashes, Sentry captures the error message, stack trace, and often a session replay — a video-like recording of what the user clicked before the crash. This is how you turn a real user bug into a reproducible test.

k6 — Load / performance testing

What it is: A tool that simulates many users hitting your API at once and measures response times (p95, p99 latency), error rates, and whether you exceeded a "budget" (e.g. checkout must stay under 350ms). You set thresholds — if p95 goes over the limit, the test fails.

In-chat artifacts — the Work Thread is the product.

Not a chat app with a terminal pasted in. A dark, artifact-rich transcript where every harness output renders as a beautiful native component — proof reel with scrubber, trace steps, visual diffs, tool calls, and rollup cards inline. Today CursorWorkThreadView is prompt + plain text; this is the target.

Design source: 05-work-thread.html wireframes + CursorArtifactCard container (shipped). Not raw log dumps — structured playback per competitor report's "Agent Work Playback": 30-second skimmable replay, jump from event → diff/log/proof.

Annotated Work Thread — all artifact types in one scroll

User prompt bubble → agent narration (short prose) → inline artifact cards. Terminal output collapsed by default. Each card is tappable for full-screen drill-down.

Proof Reel — scrubbable playback P0 visual

Short video or frame sequence tied to validation — not a Loom clone. Scrubber maps to trace steps: failing frame → fix applied → passing rerun. 30 seconds max, skimmable from the thread. Builds on RunOutputStore + trace frames.

Trace timeline + tool call cards

Playwright trace as an inline card, not a zip file. Tool calls from RunOutputStore.ToolBlock render as collapsible rows with status, duration, and expandable JSON result.

Visual diff + device matrix cards

Chromatic-style before/after and Airtable-style device grid — rendered inline, not on a separate screen. Tap tile to expand diff overlay.

Agent Work Playback — structured timeline

Flight Recorder inside the thread: phases, commands, file changes, test results, decisions — jump from any event to its artifact. Consolidates with Ship & History per master plan D11.

Mobile-native iOS features.

Harness features are content. iOS-native surfaces are why you reach for the phone instead of waiting for a laptop. Phone captures context and makes decisions; Mac runs harnesses and produces artifacts.

Second-agent review (2026-07-07): A skeptical product pass checked this list against the live repo, FEATURE_BACKLOG rejected items (Risk Meter, Haptic Risk Language), and Tier 0 gates. Verdict: strongest ideas are rich proof-backed push and Control Center emergency stop — both expose the governance/proof backend competitors can't fake. Push back hardest on swipe-to-approve (fights recent security posture) and Proof Reel (flashy, low workflow value). Use status lines like "3/4 checks passed" — not a single numeric proof score (too close to the cut Risk Meter).
Keep

Share Sheet intake, rich push + Live Activity proof line, Home widget, voice dictation, Tap-to-Isolate on static screenshots, Control Center stop (after atomic daemon fix), visual diff with tap-to-confirm.

Cut for V1

Camera OCR case file (agent already has logs), StandBy board (nightstand ≠ away-from-desk), swipe-to-approve diffs (accidental approvals), numeric proof score gauge, haptic risk encoding (rejected in backlog).

Defer

Proof Reel Stories format, on-device Foundation Models digest, full Watch app embed — build structured trace playback first.

Add (missed)

Lock-screen notification quick-reply, Shortcuts automations (intents exist), Watch complications, Focus Filters for P0-only alerts while away.

Share Sheet evidence intake P0

Only makes sense on phone. See a bug in Safari, Slack, TestFlight, or Messages → Share → Lancer creates a case file. Desktop can't capture "I saw this on my phone" in context. Already wireframed; not built.

Rich push + Live Activity proof line P0

Competitors can't copy the push — they can copy the UI. Orca/Happier/Cursor can ship approve/deny in an afternoon. Showing "3/4 checks passed, 1 visual diff needs review" requires the proof pipeline underneath. Partially built: LiveActivityPresentation, ApprovalActionIntent, push-backend Live Activity tokens.

Tap-to-Isolate annotation P1

Headline mobile differentiator. Tap the broken element on a screenshot → Vision segments it → becomes a Mission Contract line item the agent must resolve. Static screenshot only (not live camera — battery + App Store risk). Desktop equivalent: draw a rectangle. Phone: one tap.

System surfaces — stop, reply, automate

Emergency stop from anywhere. Control Center kill switch for runaway agents ($900 overnight is a documented failure mode). Gate on daemon atomic stop fix first. Plus: lock-screen quick-reply and Shortcuts automations (App Intents already exist in repo).

Thumb-native review patterns

Don't shrink desktop UIs. Design for glance + thumb: vertical step cards (not side-by-side panes), tap-to-confirm (not swipe-to-approve for irreversible actions), claim cards before code diffs.

Recommended stack (post–Tier 0)

Merged from primary research + second-agent review. Sequence: proof object first, then iOS surfaces that expose it.

P0 — first paid slice

  • Rich push + Live Activity proof status line
  • Home Screen "Needs You" widget
  • Share Sheet evidence intake
  • Voice dictation in follow-up fields
  • Minimal proof object in relay (unblocks push)

P1 — second wave

  • Control Center emergency stop (after atomic daemon fix)
  • Tap-to-Isolate on static screenshots
  • Visual diff review, tap-to-confirm
  • Notification quick-reply
  • Watch complication → glance + deny only
  • Shortcuts + Focus Filters

P2 — usage evidence only

  • On-device Foundation Models digest (summarize only, never verify)
  • Structured trace playback (before Proof Reel)
  • StandBy mission board
  • Camera OCR (cut unless hardware-debug case emerges)
Three moats competitors can't easily copy: (1) Rich push fed by real proof pipeline + policy engine, (2) Cross-vendor Control Center emergency stop with governance audit, (3) Tap-to-Isolate → Mission Contract line items. All three trace back to the governance/proof backend — not the iOS API surface alone.

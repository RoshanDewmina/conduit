Hands-free mechanics — one primitive, many surfaces

App Intent is the primitive. Siri, Shortcuts, Spotlight, widgets, Live Activity buttons, and Control Center are front ends onto the same intent.

App Intent

One action the system invokes. Build intent + entity layer once; surfaces come nearly free.

App Shortcut

Siri phrase packaging. Provider must live in app target binary — shared packages silently fail registration.

Shortcuts automations

User-composed chains. You enable by shipping good intents — geofenced dispatch is free, not a feature.

Live Activity buttons

perform() runs in widget extension — no relay session, no Face ID API. Device unlock only.

Notification actions

Proven 2026-06-23 app-closed. APNs → long-press → Approve/Deny. The current demo workhorse.

Gap today

Zero production AppEntity. "Deny the migration approval on mac-studio" can't work — Siri has nothing to refer to.

Trust & safety line

What can run unattended vs. what must open the app. Face ID is gone app-wide (2026-07-07); device unlock is the only human-presence check on unauthenticated surfaces.

Fully hands-free
Read-only: status, pending counts, what finished, find/open. Also pause (reversible).

+ Confirmation

Where Siri actually lands

Three separate things at three different times — not one "Siri feature."

1

Lock-screen loop — Layer 3

Highest-value hands-free surface. All iOS-26-safe. Blockers are Lancer bugs, not Apple gates. This is the "it just works" demo — not Siri.

2

Siri Phase 1 — Layer 3e/4

Entities + read/pause/stop/deny + voice-answer. ~1 week. Fixes real bugs. Does not move a 07-21 purchase decision.

3

Magical Siri — Layer 5

Semantic search, onscreen awareness, Siri-dispatched missions, on-device copilot. Every symbol is iOS 27 hard-gated.

Trust line holds across all three: reads go free; deny/stop/answers need confirmation; low-risk approve needs hash + tiers; everything else opens the app. Since Face ID is gone, 3d is not optional if hands-free approve ships.

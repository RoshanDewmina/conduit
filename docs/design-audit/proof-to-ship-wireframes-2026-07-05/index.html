P0 · V1 spine

Away Mode: Live Activity & Dynamic Island

Design spec approved 2026-07-05. One Live Activity per governed mission (max ~3 concurrent). Island shows highest-relevanceScore mission. Phases: needs-you → proof-ready → blocked → running. Blocked is passive (CI, hook down) — never competes with needs-you for urgency.

Cardinality

One LA per mission; multiple cards on Lock Screen; Island = winner by relevance.

Interaction

Hybrid: chips for questions, approve/reject for low/med risk, tap-through for high-risk + proof-ready.

Widget V1

Fleet small: aggregate counts + top urgent mission. Mission medium in V1.5.

Implementation

New LancerMissionAttributes; push from lancerd via relay → push-backend.

PhaseLock Screen toneIsland compactExpanded action
RunningGreen dot · mission title · agent · elapsed/costGreen dot · ...Glance only · tap opens Work Thread
Needs youAmber (red if high/critical)Dot + ? or ⚠Chips / Approve·Reject / Review decision
Proof readyLavender · gap countLavender ✓Review proof → Proof Ready page
BlockedGrey · stall reasonGrey ...No buttons · informational only

Copy rules: mission title > agent name; no secrets/diffs/paths; high-risk = whole-card tone swap; proof-ready lavender ≠ green (green implies ship).

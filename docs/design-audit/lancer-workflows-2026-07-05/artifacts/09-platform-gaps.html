Beside AI Summary card, natural voice-query response & Alipay widget galleryPlatform intelligence, all 3 panels

One card each for the three consolidated patterns: an AI-labeled summary card, a spoken answer with follow-up chips, and one widget template shown across many states in a single gallery screen.

Why consolidated, not fragmented

10+ ideas were on the roadmap as 10+ screens. They aren't 10+ screens.

The 2026-07-05 checklist verification pass named 5 separate on-device-AI ideas, 3 separate Siri ideas, and 3 separate large-portrait widget ideas (.systemExtraLargePortrait) — 11 roadmap lines that would have shipped as 11 disconnected destinations. Each group is actually one capability applied to different content: one on-device compression pattern (summary, contract draft, diff caption, narration all use the same card), one Siri answer-plus-follow-up pattern, one widget template rendered in four data states. Command & Context and Proactive Signals got the same treatment at smaller scale — three entry points into one composer/mission model, and three signal types feeding one Away Digest queue, not three new inboxes.

iOS 27 / WWDC 2026 — new findings beyond Platform Intelligence

Two capabilities relevant to the other three gaps

Platform Intelligence is already explicitly built from iOS 27 capabilities (Foundation Models, Siri App Intents, widget templates) — these two are net-new findings that apply to Command & Context, Proactive Signals, and Workspace Depth instead.

SP

Spotlight-backed local RAG for semantic jump-to

Spotlight Search Tool (Foundation Models) · IndexedEntity

Fully local retrieval-augmented generation against the device's existing Spotlight index — "no embeddings, no vector database, no setup." If the command palette or repo browser ever want semantic search ("find the file about the checkout flow") instead of purely lexical matching, indexing missions/files as IndexedEntity and querying the Spotlight RAG tool is a zero-infrastructure path there.

NG

System notification grouping now leans on urgency, not just app/time

iOS 27 notification coalescing & AI-prioritized grouping

Same-app notifications now coalesce into one updating entry, and grouping is reported to weight type/urgency over strict chronological order. Worth re-testing once dependency/security alerts and container-status pushes can arrive in the same burst — the system's own grouping could reorder Lancer's intended priority.

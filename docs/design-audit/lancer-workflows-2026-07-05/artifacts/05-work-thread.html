How someone actually uses it

One thread, start to finish

A–C

Normal path

Prompt → a plan card with a live to-do list → proof card with changed files, sticky View PR / Mark Ready.

D–E

When it needs you

A blocked patch pins an amber approval card with Review/Deny; a failed test pins a red failure card with the log collapsed by default.

F

High-risk proof

When risk is high, Verify… is the required first action on the ship rail — PR and Ready stay disabled until a cross-vendor second opinion is accepted (same proportional friction as medium+ approval review).

Artifacts

Contextual, not new roots

Question Card compresses a desktop-shaped question into 2 choices; Preview Cockpit and Tap-to-Isolate Annotation turn a proof frame into precise, sendable feedback; Command Cards give one-tap recovery on failure.

Proof

Device Matrix, Visual Diff, Auto Bug Replay

Verify across device/appearance states, compare a UI change before/after, and replay a crash's before-and-after side by side — all reached from the same proof rail.

What Mobbin taught us, and why

Delivery-app pacing, not a chat clone

Shop delivery progressSummary first

Summary always visible, detailed log collapsed in a sheet — why raw output sits behind "Show output," never the default view.

GitHub PR conversationOne durable object

Reviews, commits, and diffs attach to one work item. Why proof, approvals, and changed files all live in the same thread, not separate screens.

Manus task/agent detailArtifact density

Dense but calm detail view for one agent task — the model for Question Card and Command Cards sitting inline without becoming clutter.

Airtable status-badge tilesDevice Matrix

Small tile grid with a pass/fail badge per item — ported directly for the 4-device proof grid.

Turo before/after damage reportAuto Bug Replay

Stacked, not side-by-side, before/after — the right shape for "did the crash actually stop happening," not a static comparison.

Why this shape, not the old one

The old thread was drifting toward a chat clone

!
Chat mental model vs. V1 spec · P0, strongest IA drift in the product

Real captures showed ChatGPT-style user/assistant bubbles; the actual spec describes a phase summary + activity log + inline approvals — not a chat app. Failed turns rendered as macOS-style terminal windows with ERROR badges, reading as a shell, not governed activity. This redesign fixes the shape at the source: prose-first artifacts, terminal output collapsed by default, never the primary voice.

Two more real gaps closed by the same redesign: live runs and persisted history used to render with different chrome (reopening a thread didn't match an active one), and the interactive SSH terminal (SessionView) still exists in the tree and must stay off every V1 route.

iOS 27 / WWDC 2026 — what applies here

Vision and App Intents fit this workflow better than any other

VN

Tap-to-isolate is a real, named API

GenerateIterativeSegmentationRequest · WWDC26 Session 237

Pixel-level mask from a single tap, not a bounding box — exactly what panel C needed. A segmented region can feed straight into an on-device Foundation Models prompt for "what changed here" captioning.

DH

Device Matrix Proof has a first-party pipeline now

Unified devicectl (real device + simulator) · WWDC26 Session 260

devicectl uses one interface for physical devices and simulators this cycle — a device-matrix proof run no longer needs to special-case per device type.

AA

The risk tier gating "Verify…" has an external taxonomy

IntentAuthenticationPolicy risk tiers · WWDC26 Session 347

Apple's own Destructive/Exfiltration = High classification lines up with the same risk score that already decides when panel F's third rail action appears — one taxonomy, not two invented separately.

LA

Confirms the design, doesn't change it

Live Activity button-only model, reconfirmed unchanged · WWDC26 Session 223

No free-form text entry in a Live Activity, iOS 27 included. Question Cards and approvals correctly stay app-side for anything beyond a tap — this isn't a gap to close, it's confirmation the current design is right.

Home/Attention Redesign — Workflow 02

Scope decided with the owner for this pass: (1) enrich Home's attention card with an explicit agent name and relative age (fields the redesigned Inbox's InboxBoardCard already has); (2) give the "all clear" state a real, calm confirmation module instead of pure text; (3) fold Inbox into Home — the existing "See all" button pushes the existing InboxFeature/InboxView.swift full list, and Inbox stops being a persistent sidebar root. Explicitly out of scope: a relay/SSH status strip at the top of Home, and a "Recent decisions" audit feed below Machines — neither is proposed below.

Proposed changes

1. Attention card row anatomy

2 open questions

Current: LancerHomeView.swift approvalAttentionCard, lines 172–208; approvalSubtitle, lines 331–335.

Current behavior: header line (icon + "Approval needed"/"Approval expired" + risk badge, unchanged) followed by one subtitle line, "<tool> · <host>", then the Review/View button. No agent identity, no age.

Proposed behavior (exact row anatomy, top to bottom):

  1. Header line — unchanged: icon + title + risk badge trailing.
  2. New: small agent-initial tile (~22×22px) leading the subtitle line.
  3. Subtitle line extended from 2 to 3 segments: "<agent name> · <tool> · <host> · <relative age>". agent.displayName and relativeTimeLabel(_:) already exist in this file (lines 798–807 and 337–342) — just not wired into this card today.
  4. Review/View button — unchanged.

Informed by: Linear Mobile Inbox (avatar-tile + single dense subtitle line) and GitHub Issues (status/label/age on one line).

Open questions: (a) exact wrap/truncation behavior when all 4 subtitle segments don't fit at large Dynamic Type sizes — which segment truncates first? (b) is a 22×22 agent tile the right size relative to the existing risk badge, or should it match Inbox's 30×30 exactly for visual consistency across the two surfaces?

2. All-clear module

2 open questions

Current: LancerHomeView.swift homeHeadline, lines 465–467; connectMachineCard, lines 384–411; machinesSection, lines 346–382.

Current behavior: headline says "All clear tonight." when pendingApprovalCount == 0; below it, either connectMachineCard (no machines paired) or the live machine tree (machine paired) — no acknowledgment of "caught up" beyond the headline sentence.

Proposed behavior (exact content): a single-line, low-emphasis confirmation row inserted between the headline and YOUR MACHINES, shown whenever fleetStore.attentionItems.isEmpty: small checkmark glyph (t.ok tone) + copy — placeholder: "You're caught up — nothing needs review." No CTA on this row; the machine card/tree below it already carries the next action (pair, or open a machine). connectMachineCard itself is judged adequate as-is and does not need a rebuild — the gap was specifically the "all-clear AND a machine already paired" case (screenshot 3 above), which has zero acknowledgment today.

Informed by: Notion / Substack / Threads "all caught up" states — small checkmark, one line, no illustration, inserted inline rather than replacing the layout.

Open questions: (a) exact copy for the confirmation line (placeholder above, not final); (b) should this row appear only when a machine is already paired (to avoid redundancy with "Connect a machine"'s own copy), or unconditionally whenever all-clear?

3. Fold Inbox into Home

2 open questions

Current: LancerSidebarView.swift primaryNavigation, lines 185–211; AppRoot.swift line 1466; SidebarShellState.swift line 10.

Current behavior: the sidebar's primaryNavigation renders three rows — Home, Inbox (badge = state.pendingApprovalCount), Machines. Home's "See all" button already exists (shows when items.count > 2) and already calls onOpenInbox(), which AppRoot.swift already wires to sidebarState.navigate(to: .needsAttention) — the exact same destination the sidebar's Inbox row navigates to. .needsAttention renders the existing, already-redesigned InboxFeature/InboxView.swift.

Proposed behavior: remove the Inbox SidebarNavRow block from primaryNavigation, leaving Home and Machines as the two remaining primary rows (Governance is already folded into Settings, so this yields Home / Machines / Settings — a 3-root IA, not the 4-root IA the original audit doc proposed, since Inbox stops being a root entirely rather than becoming a 4th one). No new navigation plumbing is required — "See all" already pushes the correct existing destination and screen; the only change is removing the sidebar's persistent entry point to it.

Informed by: Linear/GitHub inbox-clarity direction from the original audit (still holds) — the fold is a navigation-IA decision, not a new visual pattern to adopt from Mobbin.

Open questions: (a) does any attention-count badge move to the Home row itself, or to the hamburger/drawer affordance, now that Inbox is gone as a sidebar row — or does the badge simply disappear from the sidebar entirely and live only on Home's own headline/cards? (b) should .needsAttention, now reached only as a drill-in from Home, get an explicit "back to Home" affordance instead of relying on generic previousDestination back-navigation?

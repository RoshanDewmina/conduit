## Notes / caveats

- This dogfood exercised the `LANCER_DEBUG_QUESTION_ANSWER` seam (HID taps are dead on this
  Simulator/Xcode-beta combination — established finding, `docs/test-runs/
  2026-07-02-device-hub-matrix-simulator-pass.md`), not a manual tap on the "Red" option. The
  underlying `RelayQuestionIngest.submit` call is identical either way (the seam drives the same
  production method the Submit button calls) — this is the same debug-seam discipline M1/M2 used,
  not a new gap.
- Only `claudeCode` was dogfooded (the only vendor `claudeStdinPromptArgv`/`--input-format
  stream-json` apply to). Codex/Kimi/OpenCode were not touched by M3 and remain on the pre-M3
  behavior (no AskUserQuestion-equivalent verified for them yet — explicitly out of scope, see the
  Plan's M4+ note).
- Physical device / APNs were not used, per this milestone's non-goals — Simulator + local
  `lancerd` + the real relay only.

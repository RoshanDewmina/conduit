# 08 — Feature opportunity ranking

> Scoring is qualitative (High/Med/Low), not a 1–10 point sum — the underlying research in `01`,
> `03`, `04`, `06`, `07` is the evidence; this table is the synthesis. "Dependency" names the
> specific blocker (an Apple API's OS gate, an owner decision, or another item in this table).

| # | Feature | Current state | User problem | Proposed experience | Apple technology | Value | Effort | Risk | Dependencies | Recommendation |
|---|---|---|---|---|---|---|---|---|---|---|
| 1 | Approval content-hash binding | Missing (`02` — no `commandHash`/`diffHash`/`toolInputHash` anywhere) | An approval isn't cryptographically tied to the exact command/diff shown — TOCTOU risk | Daemon computes a content hash at approval creation; phone echoes it back; daemon re-verifies before executing | None — pure protocol/schema work | High | Med | High (security) | None — no Apple API or OS-version gate | **Build now** |
| 2 | Risk-tiered fail-closed no-client policy | Fails open after 8s for all kinds today | An 8s reachability gap silently approves *any* action, including destructive ones | Reuse existing blast-radius classification; only low-risk kinds keep the grace period, high/critical fail closed | None | High | Low-Med | High (security) | Reuses existing Governance blast-radius work | **Build now** |
| 3 | E2E relay replay resistance | Missing (`02`, `07`) | Duplicate encrypted frames could replay a stale dispatch/run-control message | Monotonic sequence number inside the encrypted envelope + small replay cache, HMAC-bound with #1's content hash | None | High | Med | Med-High (security) | Composes with the existing connect-generation-counter fix | **Build now** |
| 4 | Fix Live Activity lifecycle (`.end()` on background) | Broken — contradicts `ARCHITECTURE.md`'s own claim (`02`, `04`) | Push-driven "while app closed" Live Activity doesn't actually work | Keep `Activity` alive on background; drive updates via push token instead of ending | ActivityKit push tokens (`iOS 16.2+`, no new OS gate) | High | Low-Med | Low | None | **Build now** |

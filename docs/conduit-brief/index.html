Approval Fatigue

Anthropic built auto mode because 93% of approvals get accepted without reading. Total bypass is dangerous. There's nothing in between.

Notification Spam

Notion: 70 agent notifications/day. Developers mute everything, then miss the one alert that mattered.

No Proof

"Task complete" in a chat bubble. No diff, no test results, no what-changed-and-why. Agents can lie and you wouldn't know.

No Cross-Agent View

Running 3 agents across 2 machines and a VPS. Each lives in a separate terminal. You have no single view of what they're all doing.

The Solution

Not "SSH on a phone, but prettier."
A mobile engineering manager.

Agents run autonomously. Conduit receives only meaningful escalations. Every completion includes verifiable proof.

🔁

Loop visibility, not notification lists

Agents report structured progress — step 4/8, blocked on test failure, waiting for decision. The fleet dashboard shows loop status, not just "active/idle."

🃏

Proof-of-work cards

Every task completion produces a structured card: goal, files changed, diff summary, tests run, risks, screenshots, "what I could not verify." Agents can't just say "done."

🛡️

Risk-tiered policy, not yes/no prompts

Approve classes of actions: allow tests, ask for package installs, block prod DB writes. Policy persists across sessions. No more per-command tapping.

📡

MCP-native — zero agent friction

Claude Code users already use MCP. Agents add one line to their config. No CLI wrapping, no SDK, no brittle shell integrations that break on every update.

🔔

Intelligent escalation filtering

The app asks "should I wake you?" before every APNs call. Most steps never reach your phone. Only blocked loops, failed tests, risky actions, and completed proofs do.

MCP Integration

Five tools. One line of config.

Agents add --mcp-server https://conduit.cloud/mcp and immediately get mobile-native loop control. No SDK. No CLI wrapping.

conduit_notify core

Fire-and-forget push to the user's device. Stored in event log. Filtered by priority and quiet hours before APNs fires.

conduit_notify({
  "message": "Build passed in 42s",
  "level": "info",
  "agent_name": "DeployBot",
  "provenance": {
    "repo": "command-center",
    "branch": "feat/mcp"
  }
})
conduit_checkpoint core

Request a human decision. Returns a checkpoint_id immediately — no 60s timeout. Fires APNs with Approve/Deny/Edit buttons.

// Returns immediately
conduit_checkpoint({
  "message": "Health check failed. Roll back?",
  "context": "2 of 3 pods unhealthy"
})
// → { "checkpoint_id": "cp_a3f2..." }
conduit_checkpoint_status

Poll until the user acts. Returns pending → approved | denied | edited. Agent loops naturally on this.

// Agent polls until decided
conduit_checkpoint_status({
  "checkpoint_id": "cp_a3f2..."
})
// → { "status": "approved" }
conduit_loop_start / conduit_step_complete new

Track structured progress. loop_start returns a loop_id and shows a progress bar in the app. step_complete notifies only on failure or block — not every step.

conduit_loop_start({
  "name": "Deploy Loop",
  "total_steps": 8
}) // → { "loop_id": "loop_9b2..." }

conduit_step_complete({
  "loop_id": "loop_9b2...",
  "step": 3,
  "status": "ok",
  "summary": "Unit tests passed (142/142)"
})
conduit_proof new

Submit structured proof-of-work. Creates a rich card in the mobile inbox. Forced schema — agents can't send vague prose. No more "task complete" with no evidence.

conduit_proof({
  "goal": "Fix failing login test",
  "changed_files": ["src/auth/session.ts"],
  "test_status": "passed",
  "diff_summary": "Fixed token refresh race",
  "risks": ["Safari OAuth not tested"],
  "recommended_next_action": "approve_pr"
})

MCP HTTP Handler

POST /mcp with JSON-RPC 2.0 dispatch. Handles initialize, tools/list, tools/call. Returns 401 for bad/missing keys.

conduit_notify + conduit_checkpoint + _status

Core tools. Notify stores event and fires APNs goroutine. Checkpoint stores pending record, fires APNs with action category, returns checkpoint_id immediately.

ready

POST /mcp-decide — iOS app calls this when user taps Approve/Deny/Edit. Updates checkpoint so polling agent sees decision.
